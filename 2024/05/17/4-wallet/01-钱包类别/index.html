钱包类别

Web3钱包的核心差异在于私钥的管理方式。私钥是用户访问和控制其加密货币资产的关键。

  • 中心化钱包：私钥由中心服务器管理，如交易所钱包（Binance、okx）
  • 去中心化钱包：私钥存储在用户设备上，如ImToken、MetaMask。采用确定性分层钱包结构，提供更高的个人控制权。
  • 硬件钱包：私钥存储在离线硬件设备中，代表产品如Ledger、OneKey，通过蓝牙、NFC或串口与用户设备通信，实现安全签名。
  • 交易所Web3钱包：集成了中心化、去中心化和硬件钱包的功能。
  • 托管钱包：托管钱包通常指的是由第三方服务提供商管理私钥的钱包。
  • 多签钱包：基于M-of-N的签名机制，要求一定数量的签名才能授权交易，如Gnosis Safe。
  • MPC钱包：采用秘密共享技术来管理和生成加密货币私钥的钱包。
  • 社交恢复钱包：两种主要的恢复机制：守护者恢复和私钥分片恢复。
  • EVM链AA钱包：遵循ERC4337协议，提供账户抽象功能，实现智能合约级别的钱包服务。

中心化钱包

架构和工作原理

  1. 私钥存储：中心化钱包的私钥通常存储在中心服务器上，由服务提供者管理。用户不直接控制私钥，而是通过账户和密码来访问自己的资产。
  2. 加密技术：为了保护私钥，中心化钱包服务提供者会采用各种加密技术，如DES加密、AES加密等，将私钥加密后存储。
  3. 签名机制：当用户发起交易时，钱包服务会解密私钥，使用私钥对交易进行签名，然后将签名后的交易发送到区块链网络。

业务流程

  1. 批量地址生成：为了提高性能和响应速度，中心化钱包会预先生成大量的地址，存放在地址池中，用户注册时直接分配地址。
  2. 充值：用户向钱包地址发送加密货币，钱包服务通过扫描区块链来确认交易，并更新用户的账户余额。
  3. 提现：用户提交提现请求，钱包服务验证请求后，使用私钥签名交易，并将加密货币发送到用户指定的地址。
  4. 归集：将分散在多个地址中的资金归集到一个或几个地址中，以便于管理和使用。
  5. 热转冷：将热钱包中的资金转移到冷钱包中，以提高安全性。
  6. 冷转热：从冷钱包中转移资金到热钱包，以便用户进行交易。
  7. 链路风控：实施风险控制措施，确保交易的安全性，防止欺诈和非法活动。

安全性

  • 内部风险：由于私钥由中心化服务管理，存在内部人员滥用权限的风险。
  • 外部攻击：中心化钱包可能成为黑客攻击的目标，一旦被攻破，用户资产将面临巨大风险。
  • 合规性：中心化钱包通常需要遵守严格的监管要求，包括KYC（了解你的客户）和AML（反洗钱）政策。

用户体验

  • 便捷性：用户不需要管理私钥，只需记住账户和密码，使用起来相对简单。
  • 服务依赖：用户的资产安全和服务可用性高度依赖于中心化钱包的运营状况。

优势与劣势

  • 优势：操作简便，适合新手用户；交易速度快，服务响应及时。
  • 劣势：用户对资产的控制权较低；存在中心化服务的风险，如内部欺诈、黑客攻击等。

去中心化钱包

也称为非托管钱包，它允许用户完全控制自己的私钥和资金，而不需要依赖于第三方服务

核心特点

  1. 私钥控制：用户完全控制自己的私钥，这是访问和转移加密货币的关键。
  2. 非托管：没有中心化的实体持有用户资产或私钥，用户对自己的资产有完全的控制权。
  3. 确定性分层钱包（HD Wallet）：支持分层确定性钱包结构，可以从一个主种子（通常是助记词）生成多个地址。

架构和工作原理

  1. 本地存储：私钥通常存储在用户的设备上，如电脑、手机或专用硬件设备。
  2. 加密保护：私钥在本地以加密形式存储，需要用户设置的密码或PIN码来解锁。
  3. 签名交易：用户发起交易时，交易会在本地设备上使用私钥签名，然后广播到区块链网络。

业务流程

  1. 生成钱包：用户创建新钱包时，系统会生成一个私钥和相应的公钥和地址。
  2. 备份助记词：生成一组助记词（通常为12或24个单词），用于恢复钱包。
  3. 接收资金：用户可以将加密货币发送到钱包的公钥地址。
  4. 发送交易：用户发起交易时，需要输入密码或PIN码来解锁钱包并签名交易。
  5. 查看余额和交易：用户可以查看钱包的余额和交易历史。

安全性

  1. 私钥安全：私钥的安全性完全取决于用户，需要妥善保管。
  2. 加密保护：私钥在本地加密存储，增加了安全性。
  3. 多签名：支持多签名钱包，需要多个私钥共同签名才能进行交易，提高了安全性。

用户体验

  1. 控制权：用户对自己的资产有完全的控制权。
  2. 隐私性：交易和余额信息不需要与任何中心化实体共享。
  3. 技术门槛：相对于中心化钱包，去中心化钱包可能需要用户具备一定的技术知识。

优势与劣势

  • 优势：用户完全控制自己的资产；提高了安全性和隐私性；支持多种区块链资产。
  • 劣势：用户需要自己管理私钥，增加了责任；可能需要更高的技术知识；交易速度可能较慢。

硬件钱包

提供高安全性的加密货币存储解决方案，专门设计的物理设备，用于生成、存储和管理私钥，以确保用户资产的安全。

核心特点

  1. 离线存储：硬件钱包将私钥存储在设备内部，与互联网隔离，只在需要时与设备交互，减少了在线黑客攻击的风险。
  2. 私钥生成：私钥在设备内部生成，确保了私钥的随机性和安全性。
  3. 用户控制：用户完全控制硬件钱包中的私钥，没有第三方介入。

架构和工作原理

  1. 嵌入式系统：硬件钱包通常基于嵌入式系统，具有有限的硬件资源，专注于安全功能。
  2. 安全元件：使用专门的安全元件（如安全处理器或加密协处理器）来保护私钥。
  3. 用户界面：配备有小屏幕和按钮，用于显示交易信息和用户操作确认。
  4. 通信接口：通过USB、蓝牙、NFC等接口与用户设备通信。

主要功能

  1. 地址生成：硬件钱包可以生成多个地址，通常遵循BIP32、BIP39等标准。
  2. 交易签名：硬件钱包在设备内部对交易进行签名，确保私钥不会暴露给用户设备。
  3. 备份与恢复：支持助记词或种子短语备份，允许用户恢复钱包。
  4. 多签名支持：一些硬件钱包支持多签名交易，增加交易的安全性。

操作流程

  1. 初始化：用户设置硬件钱包，生成新的种子短语，并妥善保存。
  2. 接收资金：用户通过硬件钱包提供的公钥地址接收资金。
  3. 发送交易：
    • 用户在计算机或手机上创建交易请求。
    • 将交易详情传输到硬件钱包。
    • 硬件钱包显示交易信息，用户确认无误后，通过按钮确认签名。
    • 硬件钱包对交易进行签名，并将签名后的交易返回给用户设备。
    • 用户将签名后的交易广播到区块链网络。
  4. 备份与恢复：在需要时，用户可以使用种子短语恢复钱包。

安全性

  1. 抵御网络攻击：由于私钥不接触网络，硬件钱包能有效抵御网络攻击。
  2. 物理安全：硬件钱包的物理安全性取决于其构建质量和用户如何保管设备。
  3. 抗胁迫功能：一些硬件钱包提供虚假的PIN码或助记词，以防止在胁迫情况下泄露真实信息。

用户体验

  1. 高安全性：为用户提供银行级别的资产保护。
  2. 易用性：尽管设置了安全措施，但大多数硬件钱包仍设计为用户友好。
  3. 成本：硬件钱包需要购买，存在一定的成本。

优势与劣势

  • 优势：极高的安全性；私钥不触网；支持多种加密货币；易于使用。
  • 劣势：需要购买硬件设备；交易时需要物理设备；可能存在设备损坏或丢失的风险。

交易所Web3钱包

核心特点

  1. 集成性：作为交易所平台的一部分，用户可以直接在交易所内进行资产的存储、转账和交易。
  2. 多功能：除了基本的钱包功能，还可能提供去中心化应用（DApp）的交互、代币交换、借贷等高级功能。
  3. 用户界面：提供用户友好的界面，便于用户管理和使用其加密资产。

架构和工作原理

  1. 中心化管理：用户的私钥由交易所管理，这与中心化钱包相似，便于交易所进行快速的交易处理。
  2. 去中心化交互：支持与区块链上的智能合约和DApp的交互，允许用户进行去中心化的交易和操作。
  3. 热钱包与冷钱包：结合热钱包（在线钱包）和冷钱包（离线钱包）的使用，以提供安全性和便捷性的平衡。

主要功能

  1. 资产管理：用户可以在钱包中查看和管理自己的加密货币资产。
  2. 交易执行：用户可以直接在钱包界面上执行买卖订单。
  3. 智能合约交互：支持与智能合约的交互，使用户能够参与到去中心化金融（DeFi）项目中。
  4. 代币收发：用户可以发送和接收各种加密货币代币。
  5. 多签支持：可能支持多签名钱包功能，增加交易的安全性。

操作流程

  1. 账户创建：用户在交易所注册账户，并创建Web3钱包。
  2. 私钥管理：私钥由交易所安全地管理，用户通过账户验证来访问钱包功能。
  3. 资金存入：用户可以通过转账将加密货币存入交易所钱包。
  4. 交易和投资：用户利用钱包内的资金进行市场交易或参与DeFi项目。
  5. 资金提取：用户可以将资金从钱包提取到个人地址或其他钱包。

安全性

  1. 中心化风险：由于私钥由交易所管理，用户需要信任交易所的安全性和信誉。
  2. 多重签名：一些交易所可能提供多重签名保护，增加交易的安全性。
  3. 冷存储：交易所通常会将大部分资产存储在冷钱包中，以抵御网络攻击。

用户体验

  1. 便捷性：用户可以在一个平台上完成交易和资产管理，无需在多个钱包间转移资金。
  2. 功能丰富：提供多种金融功能，满足用户的不同需求。
  3. 安全性：虽然不如硬件钱包安全，但通过多重签名和冷存储等措施提高了安全性。

优势与劣势

  • 优势：操作便捷，功能集成；用户界面友好；快速交易执行。
  • 劣势：中心化管理可能带来安全风险；用户对资产的控制权较低。

托管钱包

提供一种高度安全的方式来存储和管理私钥。通常由一个可信的第三方机构管理，如专业的托管服务提供商或金融机构。

核心特点

  1. 第三方管理：私钥由第三方机构管理，而不是由用户直接控制。
  2. 安全性高：通过专业的安全措施和协议来保护私钥，适用于存储大量资产。
  3. 合规性：托管钱包服务提供商通常需要遵守严格的监管要求，包括KYC和AML政策。

架构和工作原理

  1. 多签名技术：托管钱包通常采用多签名技术，即需要多个签名者中的一定数量同意才能授权交易。
  2. 分层确定性钱包结构：支持分层确定性钱包（HD Wallet），可以从一个种子短语生成多个地址。
  3. 冷存储：私钥存储在离线环境中，如硬件安全模块（HSM）或冷库，以提高安全性。

主要功能

  1. 资产保管：为用户提供加密货币资产的保管服务。
  2. 交易执行：用户可以发起交易请求，由托管服务执行实际的交易。
  3. 审计和报告：提供资产审计和交易报告服务，有助于合规性和透明度。
  4. 风险管理：通过专业的风控措施，帮助用户管理资产风险。

操作流程

  1. 注册和验证：用户在托管服务提供商处注册账户，并完成必要的身份验证。
  2. 资产存入：用户将加密货币发送到托管钱包提供的地址。
  3. 交易请求：用户通过托管服务的用户界面或API发起交易请求。
  4. 交易执行：托管服务在验证用户请求后，执行交易并将结果通知用户。
  5. 资产提取：用户可以请求将资产从托管钱包提取到指定地址。

安全性

  1. 专业安全措施：托管服务提供商采用高级的安全技术和协议，如SSL加密、多因素认证等。
  2. 保险：一些托管服务为存储的资产提供保险，以覆盖潜在的安全事件造成的损失。
  3. 合规性：遵循监管要求，定期进行安全审计和合规检查。

用户体验

  1. 高安全性：为用户提供银行级别的资产保护。
  2. 专业服务：提供专业的资产管理和交易执行服务。
  3. 监管合规：帮助用户满足监管要求，降低合规风险。

优势与劣势

  • 优势：提供高安全性的资产保管；专业的管理和服务；帮助用户满足监管要求。
  • 劣势：用户需要支付服务费用；对资产的控制权较低；需要信任第三方机构的信誉和安全性。

多签钱包

多签钱包（Multi-signature Wallets），是一种需要多个私钥中的一定数量（M个中的N个，即M-of-N）共同签署才能授权交易的加密货币钱包

核心特点

  1. 多重签名：交易需要多个私钥中的至少M个签名才能被授权。
  2. 提高安全性：减少了单点故障的风险，即使部分私钥泄露，资产仍然安全。
  3. 灵活性：用户可以根据需要设置M和N的值，以及哪些私钥参与签名。

架构和工作原理

  1. 私钥生成：每个参与方生成自己的私钥，并保留好。
  2. 公钥组合：所有参与方的公钥被组合在一起，用于生成钱包地址。
  3. 交易签署：当需要发起交易时，必须收集到足够的签名（至少M个）才能进行。
  4. 智能合约支持：多签钱包通常基于智能合约实现，特别是在以太坊等支持智能合约的区块链上。

主要功能

  1. 资产共享管理：多个用户可以共同管理钱包中的资产。
  2. 交易授权：交易需要多个签名，增加了交易的安全性。
  3. 权限分配：可以根据需要分配不同的权限给不同的私钥持有者。
  4. 审计和追踪：所有交易都需要多个签名，易于审计和追踪。

操作流程

  1. 创建钱包：创建多签钱包时，需要确定M和N的值，以及参与方的公钥。
  2. 生成私钥：每个参与方生成自己的私钥，并妥善保管。
  3. 发起交易：一个参与方发起交易请求，但交易尚未最终授权。
  4. 收集签名：交易请求被发送给其他参与方，他们用自己的私钥签署交易。
  5. 交易执行：收集到足够的签名后，交易被广播到区块链网络并执行。

安全性

  1. 抗胁迫：即使一个私钥持有者受到胁迫，也无法单独授权交易。
  2. 抗攻击：需要多个签名才能交易，增加了攻击者盗取资产的难度。
  3. 私钥分散：私钥分散在多个参与方手中，减少了私钥集中泄露的风险。

用户体验

  1. 安全性高：适合需要高安全级别的用户和组织。
  2. 操作复杂性：相比于单签名钱包，多签钱包的操作更为复杂。
  3. 协作需求：需要参与方之间的协作来签署和执行交易。

优势与劣势

  • 优势：提高了安全性和抗风险能力；适合共同管理资产；易于审计和追踪。
  • 劣势：操作过程可能较为繁琐；需要多个参与方的协作；可能存在协调困难。

MPC钱包

（MPC，Multi-Party Computation），一个完整的私钥被分割多个部分，每个部分由不同的参与方持有。

核心特点

  1. 去中心化控制：私钥以分割的形式存在，没有单一实体能够单独控制私钥。
  2. 增强安全性：私钥的每个部分单独不具有价值，只有组合一定数量的部分才能恢复私钥。
  3. 多方授权：需要多个参与者的份额共同协作才能授权和签署交易。

架构和工作原理

  1. 秘密分割：私钥通过秘密共享算法被分割成多个份额，每个份额包含私钥的一部分信息。
  2. 份额分布：这些份额被分发给不同的参与者，可以是不同的个人、服务器或地理位置。
  3. 阈值设置：设置一个阈值（通常是N个中的M个），只有收集到M个份额时才能恢复私钥。
  4. 交易签署：当需要签署交易时，所有参与者必须协作，提供他们的份额以生成签名。

主要功能

  1. 资产保护：即使部分份额丢失或被盗，只要没有达到阈值，资产仍然安全。
  2. 抗胁迫：单个参与者无法被胁迫透露私钥信息，因为他们不拥有完整的私钥。
  3. 容错性：即使一些份额丢失或损坏，只要有足够的份额，私钥仍然可以被恢复。
  4. 审计和合规：所有交易都需要多个签名，便于审计和满足合规要求。

操作流程

  1. 初始化钱包：创建MPC钱包时，私钥被分割成多个份额，并分配给不同的参与者。
  2. 份额存储：每个参与者安全地存储他们的份额，可能使用硬件安全模块(HSM)或其他安全措施。
  3. 交易请求：用户发起交易请求，需要多个份额来授权。
  4. 份额收集：协调者或智能合约收集足够的份额以重构私钥。
  5. 签名交易：使用重构的私钥对交易进行签名。
  6. 执行交易：将签名后的交易广播到区块链网络。

安全性

  1. 抗盗取：没有单一实体拥有完整的私钥，增加了盗取资产的难度。
  2. 抗内部欺诈：需要多个参与者的协作才能进行交易，减少了内部欺诈的风险。
  3. 抗胁迫：单个参与者即使在胁迫下也无法泄露完整私钥。

用户体验

  1. 高安全性：用户可以放心地保管资产，因为私钥的安全性得到了显著提升。
  2. 操作复杂性：相比于传统钱包，MPC钱包的操作可能更加复杂，需要协调多个参与者。
  3. 信任和协作：需要在参与者之间建立信任和协作机制。

优势与劣势

  • 优势：提供了极高的安全性；适合需要严格合规的场合；增强了对资产的保护。
  • 劣势：操作过程可能较为复杂；需要多个参与者的协作；可能涉及较高的协调成本。

社交恢复钱包

守护者恢复机制

在守护者恢复机制中，用户可以指定一组信任的联系人（守护者），他们可以在用户丢失私钥时帮助恢复钱包。这个过程通常涉及以下步骤：

  1. 设置守护者：用户在钱包中设置一组信任的联系人（守护者），通常需要他们的公钥或钱包地址。
  2. 私钥丢失：当用户丢失了对钱包的访问权限，例如忘记了密码或丢失了硬件钱包。
  3. 请求恢复：用户向守护者发出帮助恢复的请求。
  4. 多签名交易：守护者们需要按照预设的规则（例如，需要一定数量的守护者同意）签署一个特殊的多签名交易，以证明他们的身份和同意恢复操作。
  5. 恢复钱包：一旦收集到足够数量的签名，就可以通过智能合约或其他机制来恢复钱包的访问权限。

私钥分片恢复机制

私钥分片恢复机制则是一种基于秘密共享算法的方法，它将私钥分割成多个部分（分片），并将这些分片分散存储在不同的地点或人那里。以下是私钥分片恢复的一般步骤：

  1. 私钥分片：用户的私钥被分割成多个分片，每个分片都通过某种加密方法进行保护。
  2. 分片存储：这些分片可以存储在用户的多个设备上、云端服务、或者交给信任的朋友保管。
  3. 分片恢复：当用户需要恢复私钥时，他们需要收集一定数量的分片（根据设定的阈值，例如5个分片中的3个）。
  4. 重组私钥：收集到足够的分片后，用户可以通过一个解密过程重组原始的私钥。
  5. 访问钱包：一旦私钥被重组，用户就可以重新访问他们的钱包。

社交恢复钱包的优势和挑战

  • 优势：提供了一种不依赖于单一备份的私钥恢复方式，增加了私钥管理的灵活性和安全性。
  • 挑战：需要用户信任守护者不会滥用他们的私钥信息，同时也要求用户在社交网络中有一定的信任基础。

EVM链AA钱包

核心特点

  1. 智能合约钱包：AA钱包本身就是部署在区块链上的智能合约。
  2. 内置安全性：通过智能合约的代码可以实施复杂的安全策略。
  3. Gas代付：允许用户在执行交易时由第三方代付Gas费用。

架构和工作原理

  1. 账户抽象：与传统钱包不同，AA钱包将账户和交易的发送者抽象化，允许更灵活的交易发起和签名方式。
  2. 交易构造：交易的构造可以在链上完成，用户或第三方可以代表AA钱包构造交易。
  3. 签名机制：AA钱包可以使用多种签名机制，包括但不限于传统的ECDSA签名。

主要功能

  1. 代付交易：支持Gas代付功能，允许第三方支付交易费用。
  2. 自定义交易：用户可以自定义交易属性，如nonce、gas限制等。
  3. 多签交易：可以集成多签名机制，提高交易的安全性。
  4. 智能合约交互：作为智能合约，AA钱包可以与其他智能合约进行交互。

操作流程

  1. 部署钱包：在以太坊链上部署AA钱包智能合约。
  2. 资金存入：用户将资金存入AA钱包合约地址。
  3. 交易构造：用户或第三方构造交易，包括确定接收者、金额和数据。
  4. 交易签名：根据AA钱包的配置，可能需要用户或其他参与方的签名。
  5. 交易执行：将构造好的交易发送到区块链网络，由网络中的节点验证和执行。

安全性

  1. 智能合约安全：安全性依赖于智能合约代码的健壮性，需要经过严格的安全审计。
  2. 多签支持：如果集成多签名机制，可以增加交易的安全性。
  3. 自定义安全策略：可以通过智能合约实现自定义的安全策略和检查。

用户体验

  1. 灵活性：提供了高度灵活的交易构造和签名方式。
  2. 降低成本：通过Gas代付功能，可以降低用户的交易成本。
  3. 复杂性：相比于传统钱包，AA钱包的设置和使用可能更复杂。

优势与劣势

  • 优势：提供更高的安全性；支持Gas代付；允许灵活的交易构造和签名方式。
  • 劣势：需要用户理解智能合约的概念；可能存在智能合约安全风险。
