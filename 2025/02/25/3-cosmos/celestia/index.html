1. 桥节点（Bridge Node）- 数据中继引擎

核心职责：连接共识层（Celestia-Core）与数据可用性（DA）网络

工作流程

1. 连接核心链

  • 通过 gRPC/RPC 直连 Celestia-Core 节点（验证节点或全节点）
  • 监听 NewBlock 事件：core.Subscribe("NewBlock")

2. 数据扩展处理

3. 广播机制

  • 使用 GossipSub 协议的 HeaderSub 主题广播扩展头
  • 消息结构：{Height, Hash, DAH_Root, ValidatorSet}
  • 网络拓扑：桥节点形成 DA 网络的初始数据源

4. 关键特性

  • 无状态设计：不存储历史数据
  • 故障恢复：自动重连核心链
  • 负载均衡：可连接多个核心节点

2. 全节点（Full Node）- 数据完整性卫士

核心职责：保证数据完整性与生成欺诈证明

头部同步

  • 订阅 HeaderSub 获取扩展头
  • 使用 HeaderSync 服务维护头部链：headerChain.Add(extHeader)

数据重建

欺诈证明生成

存储结构

3. 轻节点（Light Node）- 可验证的轻客户端

核心职责：通过随机采样低成本验证数据

头部同步优化

  • 初始同步：从可信检查点开始
  • 增量更新：订阅 HeaderSub 获取最新头
  • 信任机制：设置 trustPeriod = unbondingPeriod

数据可用性抽样（DAS）

欺诈证明处理

信任模型

  • 初始信任：依赖创世或可信检查点
  • 持续验证：通过 DAS 建立概率信任
  • 故障熔断：欺诈证明触发安全停机

4. 跨节点协作场景

  • 新区块生命周期
  • 轻节点同步流程
    • 启动时获取可信检查点（高度=H）
    • 从 HeaderSub 接收最新头（高度=H+100）
    • 计算信任间隔：currentTime - header[H].Time
    • 若超过 trustPeriod，需请求新可信头，重新建立信任链
    • 并行执行：HeaderSync 继续同步头部，DASer 验证历史数据可用性

5. 核心技术实现

数据可用性头（DAH）结构

NMT树优化

  • 去除17字节前缀，通过树结构推导节点类型
  • 命名空间存储分离，不重复存储 namespaceID

网络协议栈

协议层 功能
FraudSub 欺诈证明传播
HeaderSub 扩展头广播
ShareNet 分片数据传输
TxSub 用户交易中继

6. 安全机制深度

长程攻击防护

  • 信任期（trustPeriod）= 解绑期（unbondingPeriod）
  • 主观头过期时触发重新信任
  • 基于 Tendermint 轻客户端模型

抽样安全保证

  • 采样30个分片可检测99.9%的数据不可用
  • 公式：1 - (1 - missing_ratio)^samples

经济安全模型

  • 欺诈证明使作恶成本显性化
  • 无效证明发送者会被信誉系统惩罚
  • 节点停机保护用户资产安全

7. 总结

Celestia 通过分层架构实现：

  • 解耦验证：共识与数据可用性分离
  • 横向扩展：轻节点可无限扩展
  • 最小信任：轻客户端不依赖全节点诚实性
  • 成本优化：存储负担由全节点承担

关键创新点在于通过密码学和经济机制设计，使轻节点能以极低成本验证超大规模数据集的可用性，为模块化区块链奠定基础。
