永续合约：机制与实现

一、业务需求与市场背景

1. 市场需求

  • 规避到期日烦恼
    传统期货合约有固定交割日，投资者需频繁展期，带来滑点、手续费和管理复杂性。
  • 无限期持仓需求
    市场需要一种可无限期持有、具备杠杆特性的工具，便于长期对冲和趋势投资。
  • 加密货币市场特性
    加密市场波动大、产品迭代快，传统交割机制复杂，永续合约更契合加密生态。
  • 高流动性工具
    永续合约通常是交易所最活跃的产品，吸引套利、投机、对冲等多类参与者。

2. 核心作用

  • 无期限杠杆化敞口：满足用户长期杠杆做多/做空需求。
  • 价格发现：反映市场对未来价格的预期，参与价格形成。
  • 风险管理（对冲）：现货持有者可对冲下跌风险，矿工可锁定收益。
  • 套利工具：现货、永续、其他衍生品间套利。
  • 高效投机工具：放大收益与风险，提升资本利用率。

3. 关键目标

  • 锚定：确保合约价格紧跟现货指数价，是永续合约的核心。
  • 消除强制到期/展期：无需定期移仓。
  • 资本效率：保证金、自动减仓等机制提升杠杆与风险控制。

二、核心机制与技术组件

永续合约通过机制（非交割日）实现价格与现货趋同，核心是资金费率机制。

1. 核心机制：资金费率

  • 原理：定期（如每8小时）在多头和空头间转账。
  • 驱动力：
    • 合约价 > 现货指数价（正溢价）：多头付费给空头。
    • 合约价 < 现货指数价（负溢价）：空头付费给多头。
  • 经济激励：
    • 合约价高，多头持续付费，促使多头平仓或吸引空头，价格回归。
    • 合约价低，空头持续付费，促使空头平仓或吸引多头，价格回归。
  • 计算要素：
    • 溢价指数（P）：合约价与现货指数价的偏离百分比（平滑处理）。
    • 利率成分（I）：小的固定值或基准利率，确保合约价与指数接近时也有净支付。

  • 结算方式：仅结算时持仓用户参与，直接划转账户余额，不影响开仓价和未实现盈亏。

2. 关键系统组件

  • 交易引擎：撮合订单，更新盘口，需高性能。
  • 风险管理系统：
    • 保证金计算：实时计算初始/维持保证金、盈亏、余额。
    • 强平引擎：监控保证金率，触发强平，处理穿仓。
    • ADL引擎：自动减仓盈利最多的对手方，承接穿仓仓位。
  • 资金费率系统：
    • 数据采集：收集现货数据，计算指数价。
    • 标记价格：基于现货指数和合约深度计算，减少操纵影响。
    • 溢价指数与资金费率计算：定期采样，结算前取平均，提前公告。
    • 结算执行：定时扫描持仓，按费率划转资金。
  • 预言机系统：从多家交易所获取现货数据，校验、过滤异常，抗攻击。
  • 清算系统：处理资金费、手续费、资产划转，确保原子性。
  • 账户系统：管理用户资产、仓位、历史。
  • API Gateway & WebSocket：提供交易、行情、账户等接口。
  • 行情系统：广播盘口、成交、标记价、K线等数据。
  • 数据库：存储用户、订单、成交、仓位、资金、费率历史等数据。

三、业务流程示例

1. 用户视角

  1. 下单：用户下达买入1 BTC永续合约（做多）限价单。
  2. 撮合：订单进入买单队列，触及限价或市价单成交。
  3. 建仓：账户显示持有1张BTC永续多单，展示开仓价、标记价、未实现盈亏。
  4. 资金费率作用：系统持续计算溢价和资金费率，假设标记价高于指数价，资金费率为正。
  5. 结算：结算时，多头支付资金费，空头获得资金费。
  6. 持仓变化/平仓：用户可加仓、平仓。
  7. 强平监控：若亏损大导致保证金率触及维持线，触发强平，ADL介入。
  8. 平仓/交割：主动或被动平仓，盈亏计入账户。

2. 后台视角（资金费率结算为例）

  1. 预言机采集现货价格。
  2. 资金费率系统：
    • 定期计算现货指数价（多交易所加权）。
    • 定期计算标记价（基于指数和基差）。
    • 定期计算瞬时溢价 P_inst = (MarkPrice - IndexPrice) / IndexPrice。
    • 结算窗口内取平均溢价 P_avg。
    • 计算资金费率 F = P_avg + I，结算前公布。
  3. 结算执行器：
    • 定时（如UTC 0:00、8:00、16:00）启动。
    • 扫描所有持仓，按方向和名义价值结算资金费。
    • 资金划转，记录结算流水。

四、核心业务逻辑

1. 价格锚定逻辑

  • 计算并广播现货指数价和标记价。
  • 根据两者差异计算资金费率，定期结算，经济激励促使价格回归。
  • 公式：

    资金费率 = 溢价指数平均值 + 固定利差

2. 杠杆与保证金逻辑

  • 初始保证金率：开仓所需最低保证金比例（如5%即20倍杠杆）。
  • 维持保证金率：维持仓位不被强平的最低比例（低于初始保证金率）。
  • 保证金计算：
  • 强平逻辑：保证金率低于维持线时，触发强平。

3. 强平与ADL逻辑

  • 强平顺序：取消挂单 → 按标记价计算破产价 → 市价平仓 → 若穿仓则ADL介入。
  • ADL机制：自动减仓盈利最多的对手方，优先弥补穿仓损失，剩余盈利归被减仓用户。

4. 资金费率结算逻辑

  • 定时扫描持仓用户，按预定资金费率和仓位方向结算。
  • 资金自动划转，仅影响可用余额，不改变仓位或开仓成本。

5. 流动性管理逻辑

  • 盘口深度：决定滑点和强平时的承接能力。
  • 手续费模型：Maker（挂单）费率低或返佣，Taker（吃单）费率高，鼓励流动性。
  • 爆仓单处理：强平以市价单发出，良好流动性可降低强平冲击。
